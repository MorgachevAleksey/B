Типы данных

Типизация rust - статическая
Все типы должны быть определены на этапе компиляции
Обязательно прописать тип вручную если в резулььтате метода или литерала может присвоится несколько типов

Подразделяются на:
- Скалярные
- Составные

Скалярные типы данных:
- Целочисленный
- Числа с плавающей
- Логический
- Символы

Целочисленный

i - со знаком (используется дополнительный код)
u - без знаков
цифра после - колличество бит хранения 8,16,32,64,128
например - u32, кстати самый ходовой (< 4.2 млрд)
isize и usize - архитектурно-зависимые, растут вместе с разряднойстью процессора: 32-4, 64-8, 128-16, нужны для индексакции каких-либо коллекций

Переполнение

Поведение при переполнение при запуске с отладкой и без - разное
1. При запуске с отлдакой - при переполненнении программа запаникует 
Механизм паники будет рассотрен далее
2. При запуске release - обертывание второго дополнения
Грубо говоря - в переменной окажется остаток от деления на (максимальное значения типа + 1)

Для обработки переполнения:
- wrapping_*
- checked_*
- overflowing_*
- saturating_*

Числа с плавающей точкой:
f32, f64 (по-умолчанию f64)
оба знаковые
соответсвуют IEEE-754
f64 работает почти так же быстро как f32, но точнее (двойная точно против одинарной)

Целочисленное деление округляет вниз, до ближайшего целого
